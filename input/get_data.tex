\section{Aquisição dos dados para identificação do sistema}
\label{sec:get_data}
%===============================================================================

O sistema apresentado na seção (\ref{sec:modelling}) é a descrição matemática para
a planta do sistema, e a função de transferência utilizada, é a que descreve a
posição angular do sistema com relação a tensão aplicada sobre os terminais do motor.

Na prática o que o que foi utilizado é um sistema como o apresentado na Figura (\ref{fig:sys_pid})
onde a planta que se quer identificar esta sendo controlada por um controlador do tipo PID
(Proporcional, integral diferencial) \cite{Ogata}. Onde aplica-se uma referencia a o sistema 
deve seguir esta referencia.

\begin{figure}[htbp]
	\center
	\includegraphics[width=0.98\columnwidth]{figures/sys_pid.eps}
	\caption{Representação do sistema utilizado para a coleta dos dados a serem utilizados no 
	processo de estimativa dos parâmetros da função $G(q)$.}
	\label{fig:sys_pid}
\end{figure}

Com este sistema foi possível fazer coleta dos dados dos sinais $r(t)$, $y(t)$ além do sinal
$u(t)$ que é o sinal de entrada da planta a ser identificada.

Com estes dados é possível proceder de duas maneiras distintas para estimar os valores dos 
parâmetros de $G(q)$, é possível apenas utilizar os sinais $r(t)$ e $y(t)$ e adicionar ao modelo
apresentado na seção (\ref{sec:modelling}) o modelo do controlador PID, já que este tem um modelo
conhecido e seus parâmetros foram previamente escolhidos (antes da simulação).

Outra maneira possível é utilizar apenas os sinais $u(t)$ e $y(t)$, desta forma pode-se ignorar a
existência do controlador PID, e utilizar o método desejado, apenas com o modelo do processo.
